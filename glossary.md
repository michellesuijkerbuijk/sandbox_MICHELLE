commit = a data object which contains the changes made to the repository, the author of the changes, and the time+date of the changes. It has a unique identifier in SHA format, which can be used to refer back to this specific commit. It also includes a 'message': a description of the changes made in a short format.
version control = a system that records changes to a file or set of files over time so that you can recall specific versions later
branch = an independent line of development within a Git repository. Each branch can contain its own commits and changes without affecting other branches
